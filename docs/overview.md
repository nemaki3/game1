# clickergame1 - プロジェクト概要

## プロジェクトの流れ

このプロジェクトは、友人と共同でブラウザゲームを開発・公開することを目的としてスタートしました。
ゲームのコードは Claude（AI アシスタント）に作成してもらい、GitHub Pages を通じてインターネットに公開しています。

### 構築の手順

| ステップ | 内容 |
|---------|------|
| 1 | GitHub アカウント作成 |
| 2 | WSL に SSH キーを作成し GitHub に登録 |
| 3 | GitHub にリポジトリ（game1）を作成 |
| 4 | WSL からリポジトリをクローン |
| 5 | Claude にゲームの `index.html` を作成してもらう |
| 6 | `git add` → `git commit` → `git push` で GitHub にアップロード |
| 7 | GitHub Pages の設定でインターネットに公開 |

## ゲームの仕組み

### ゲーム概要

「clickergame1」はシンプルなクリッカーゲームです。
ボタンをクリックするとスコアが増え、貯まったスコアでアップグレードを購入すると
1クリックあたりの獲得スコアが増えていきます。

### 使用技術

| 区分 | 技術 | 説明 |
|------|------|------|
| マークアップ | HTML5 | ゲーム画面の構造を定義する言語 |
| スタイル | CSS3 | 色・レイアウト・アニメーションなど見た目を担当 |
| ロジック | JavaScript (バニラ JS) | クリック処理・スコア計算・アップグレード処理などゲームの動作を担当 |
| ホスティング | GitHub Pages | GitHub のリポジトリから静的ファイルをそのまま公開できる無料サービス |

> **バニラ JS とは**: 追加のライブラリやフレームワークを使わず、ブラウザ標準の JavaScript だけで書かれたコードのこと。シンプルで学習コストが低い。

### ファイル構成

```
game1/
├── index.html      # ゲームのすべて（HTML・CSS・JS）が入ったメインファイル
└── docs/
    └── overview.md # このドキュメント
```

現在はすべてを `index.html` 1ファイルにまとめています。
ゲームが複雑になってきたら `src/` フォルダに分割していく予定です。

### index.html の内部構造

```
index.html
├── <head>
│   └── <style>  ← CSS（見た目・レイアウト）
└── <body>
    ├── HTML     ← 画面の要素（ボタン・スコア表示・アップグレード一覧）
    └── <script> ← JavaScript（クリック処理・スコア管理・アップグレード処理）
```

### ゲームロジックの概要（JavaScript）

```
1. クリックボタンを押す
   → score += clickPower（現在の倍率分スコアを加算）
   → スコア表示を更新
   → アップグレードボタンの有効/無効を再チェック

2. アップグレードボタンを押す（スコアが足りる場合）
   → score -= cost（購入コストを差し引く）
   → clickPower += power（クリック倍率を加算）
   → ボタンを「購入済み」として無効化
```

### アップグレード一覧

| アップグレード | コスト | 効果 |
|--------------|--------|------|
| クリック強化 Lv.1 | 10点 | +1点/クリック |
| クリック強化 Lv.2 | 50点 | +5点/クリック |
| クリック強化 Lv.3 | 200点 | +20点/クリック |
